Práctica de proyecto front-end

Nos gustaría poder encontrar los sitios en Barcelona de una manera más fácil por eso necesitamos crear una aplicación que nos facilíte la tarea utilizando la api Places de google.

Funcionalidad

Por un lado, disponemos de un buscador de sitios. Al introducir un término de búsqueda en el formulario y pulsar intro o el botón de buscar, deben aparecer los sitios en el listado inferior.

Por otro lado, disponemos de unas categorías en el menu lateral. Al clicar sobre una categoría deben aparecer los sitios de esta categoría en el listado de al lado.

Al pulsar sobre un sitio, debe desaparecer el listado de sitios y aparecer la información sobre el sitio (en el archivo index.html es la sección de ‘Places details’): Imagen, Nombre del sitio, Dirección del sitio, Teléfono, Puntuación, si esta abierto o Cerrado, el sitio web. En caso de que el sitio tenga comentarios estos deben mostrarse en la sección “Place reviews”

El formato con el que se ha de mostrar la información es igual al del ejemplo que hay precargado en la página. La página debe limpiarse para que al abrirla el formulario esté limpio, aparezca las categorías y no aparezca ningún listado.

API

Para obtener la información necesaria deberemos utilizar varios endpoints de la API de Google Places:

Los Ids de categorías son: restaurant, bakery, bar, museum, gym, supermarket

  • Listado de categoría: https://developers.google.com/maps/documentation/javascript/places#place_search_requests 
let request = {
    location: new google.maps.LatLng(41.411549, 2.194118), // Barcelona
    radius: 50000, // 50 Km keyword:
    'supermarket', //restaurant, bakery, bar, museum, gym, supermarket
}
service.nearbySearch(request, resolve);
  • Búsqueda por nombre: https://developers.google.com/maps/documentation/javascript/places#place_search_requests 
let request = {
  location: new google.maps.LatLng(41.411549, 2.194118), // Barcelona
  radius: 50000, // 50 Km
  keyword: '100 montaditos', // Nombre a buscar
}

service.nearbySearch(request, resolve);
  • Información de un sitio: https://developers.google.com/maps/documentation/javascript/places#place_details 
let request = {
  placeId: "PLACE_ID",
  fields: [
    'formatted_address',
    'name',
    'rating',
  ]
}

service.getDetails(request, resolve);

Restricciones

Los archivos necesarios para realizar la práctica están en el archivo app.zip. El archivo que se utilizará para probar la práctica está en app/dist/index.html.

Todo el código JavaScript debe estar escrito utilizando módulos (tantos como sea necesario). Se pueden utilizar paquetes externos de npm siempre que esté debidamente justificado (si realmente es necesario y si compensa en términos de funcionalidad aportada y tamaño adicional en el archivo generado).

La aplicación debería funcionar bien en navegadores modernos. En caso de dudar si una funcionalidad está disponible puede utilizarse la web caniuse.

Todo el código del proyecto debe combinarse en un único archivo JavaScript optimizado (app/dist/js/app.min.js), ejecutando el comando npm run build desde la carpeta “app”.

El archivo HTML sólo puede modificarse para eliminar el código de ejemplo, añadir identificadores para poder referenciar el código fácilmente desde JS y, opcionalmente, para añadir las plantillas realizadas a a partir del código de ejemplo.

IMPORTANTE: No se debe enviar la carpeta “node_modules”. Todo su contenido se debería generar automáticamente al ejecutar npm install.
